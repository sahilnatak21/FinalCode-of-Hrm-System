## Quick Setup (3 Steps)

### 1. Build Backend
```bash
cd HRM_SYSTEM_BACKEND
mvn clean install
mvn spring-boot:run
```

### 2. Create Test Users
```powershell
.\setup-login-users.ps1
```
Or with curl:
```bash
curl -X POST http://localhost:8080/api/admin/create-user \
  -H "Content-Type: application/json" \
  -d '{"username":"hr_admin","password":"password123","role":"HR"}'
```

### 3. Test Login
- URL: `Frontend/login.html`
- Username: `hr_admin`
- Password: `password123`
- Role: `HR`
- Complete CAPTCHA

## API Endpoints

### Public (No Auth Required)
```
GET  /api/auth/captcha                    → Get login CAPTCHA
POST /api/auth/login                      → User login
```

### Admin Setup (Use Initially)
```
POST /api/admin/create-user               → Create new user
POST /api/admin/migrate-passwords (once)  → Hash existing passwords
```

## Error Messages

| Error | Cause | Fix |
|-------|-------|-----|
| "Captcha expired" | Too much time passed | Click Refresh button |
| "Invalid captcha" | Wrong CAPTCHA entered | Re-read and enter correctly |
| "Invalid username or password" | Wrong credentials | Check username/password |
| "Invalid role for this user" | Wrong role selected | Use correct role for user |
| "Username and password are required" | Empty fields | Fill all required fields |
| "Please select a role" | No role selected | Select HR or Employee |

## Troubleshooting Flowchart

```
❌ Backend won't start?
├─ Check MySQL is running (localhost:3306)
├─ Verify database credentials in application.properties
└─ Run: mvn clean install

❌ Login fails with invalid credentials?
├─ Verify user created with /api/admin/create-user
├─ Check username/password are exact match
└─ Try test user: hr_admin / password123 / HR

❌ CAPTCHA shows "Captcha unavailable"?
├─ Verify backend is running
├─ Check browser console (F12) for network errors
└─ Try refreshing the page

❌ Login succeeds but doesn't redirect?
├─ Check hr-dashboard/index.html and employee-dashboard/index.html exist
├─ Open browser console (F12) to see any errors
└─ Verify relative paths are correct
```
